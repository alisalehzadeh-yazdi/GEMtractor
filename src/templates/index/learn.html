Which type of models does the enalyzer support?

The enalyzer supports only models encoded using the Systems Biology Markup Language (SBML) format. SBML is a standard format for computer models of biological processes, based on XML. You can find many example models at Biomodels or at BiGG Models.

How does the enalyzer identify genes that are associated to a reaction?

We currently support two formats of gene associations: The legacy SBML-notes annotations, and well-structured annotations using the FBC package in SBML Level 3 documents.

An example of legacy SBML-notes annotations from MODEL1011080003, which encodes for 438 or 330: 

<reaction id="R_METSR_S1" [...]>
	<notes>
		<html:p>GENE_ASSOCIATION: ( 438  or  330 )</html:p>
		<html:p>PROTEIN_ASSOCIATION: MsrA</html:p>
		<html:p>SUBSYSTEM: S_Amino_Acid_Metabolism</html:p>
		<html:p>PROTEIN_CLASS:</html:p>
	</notes>
	[...]
</reaction>

Why do genes have such a strange name?

The gene names used by the enalyzer are actually the gene's identifiers, typically generated by some kind of software. They are often just numbered with a optional prefix (such as g1, g2, g3 etc). See also What if no gene is associated to a reaction?

What if no gene is associated to a reaction?

If the enalyzer cannot find a gene-association, it assumes there is only a single gene and gives it the id of the reaction. See also How does the enalyzer identify genes that are associated to a reaction?

Which data does the enalyzer store on the server?

If accessed through a webserver, it stores the uploaded model (or the model downloaded from bigg or biomodels) and some session related settings. To see what the enalyzer stores about you and how to delete the data it stores head over to the privacy policy.

Why am I always redirected back to model selection?

You probably do not accept cookies..? The enalyzer uses sessions to store your settings. These sessions rely on a cookie in your browser. If the enalyzer doesn't know which model you selected, it won't let you do for filtering or exporting... So you'll always be redirected to model selection.

To learn more on what's stored in your session have a look into the privacy policy.

How can I modify the default settings?

The settings of the enalyzer are controlled through environment variables on the server. Thus, you can only change the settings, when running your own enalyzer server! Just scroll down to learn how to run your own enalyzer — it's pretty easy.

Use the enalyzer API

Send Job as JSON

The API is listening at URL/api/execute — it expects a job encoded as JSON and sent via HTTP POST, such as this: 

{
  "export": {
    "network_type":"rn",
    "network_format":"sbml",
    "remove_reaction_genes_removed": False,
    "remove_reaction_missing_species": True
  },
  "filter": {
    "species": ["h2o", "atp"],
    "reactions": [],
    "genes": ["gene_abc"],
  },
  "file": "<sbml>...</sbml>"
}

Keys of the job's JSON object

export

The key export is required, and it must encode for a JSON object containing the following keys:

  • network_type: String "en" or "rn" — do you just want the filtered reaction network ("rn"), or should the enalyzer return the extracted the enzyme network ("en")?
  • network_format: String "sbml", "graphml", "gml", or "dot" — how should the returned network be encoded?
  • remove_reaction_genes_removed: Boolean — remove reactions, whose genes were all removed? (Optional, defaults to true)
  • remove_reaction_missing_species: Boolean — remove reactions, in which at least one species was removed? That is, if you remove a species from a reaction, should the enalyzer also remove the whole reaction? (Optional, defaults to false)
Scroll up to learn more about the export options.

filter

The key filter is optional. You just need to provide it, if you want to get rid of some items in the model. If provided, it may contain the following keys:

  • species: list of species-identifiers to remove from reactions
  • reactions: list of reaction-identifiers to remove from the model
  • genes: list of gene-identifiers to remove from reactions
Every list element is supposed to be a string, which corresponds to the id attribute of the SBML entity. However, every list can also be empty.

file

The key file is required. It must contain the raw SBML code of your input model, including the <?xml version="1.0" encoding="UTF-8"?> prefix.

Expect a response

Reading, filtering, converting, and exporting may of course take some time, but if the enalyzer finishes successfullty it will return with an HTTP response status 200 and immeditaly serve the exported content.

If the enalyzer was not successful, you'll get an HTTP status other than 200, for example:

  • 400: Your request was bad, for example the JSON was not well formed or is missing a required key, or the supplied model is not valid SBML.
  • 302: You forgot to send the job as POST data. You'll, thus, be redirected to this section to lear how to fire a job :)
  • 500: There was a problem on the server, eg. we could not generate the desired file.
The responded content may give you some hint on what went wrong.
